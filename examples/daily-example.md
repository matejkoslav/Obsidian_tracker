# Daily Template Examples

Use this file as a reference. Do not use it as the daily template unless you want examples copied into every daily note.

---

## 🚀 Work Output

### What moved the needle today?

Write real output, not effort.

Good examples:

- shipped feature
- commit / PR
- deployed something
- sent outreach
- published post
- finished concrete deliverable
- made money / got lead / got response
- fixed real bug
- completed school task that actually had deadline/value
- recorded/sent/published something
- made a decision that unlocks execution

Bad examples:

- watched tutorial
- planned stack
- tweaked colors
- reorganized notes
- read docs without applying
- thought about project
- researched tools for too long
- asked AI for another plan without executing

## 🧨 Fake work / time leaks

Write things that looked productive but did not create real output.

Examples:

- tweaking UI instead of shipping
- reading docs too long
- changing tools
- overthinking stack
- planning without execution
- watching tutorials instead of building
- refactoring without clear reason
- checking AI/model/tools instead of doing task
- browsing “best setup” content
- doing low-impact admin first
- rewriting tasks instead of doing them
- making the system prettier instead of using it
- researching productivity instead of producing output

## 🔁 One change for tomorrow

One concrete change. Not a list. Not vague motivation.

Good examples:

- First 60 min = coding only, no reading.
- Write docs for 20 min before opening YouTube.
- Ship ugly version before tweaking design.
- Marketing post before checking tools.
- One outreach before any planning.
- Start with the task I avoided today.
- No tool/setup changes before one real output.
- Limit docs reading to 20 min, then implement.
- If stuck for 15 min, write the blocker and next action.

Bad examples:

- be more productive
- focus more
- waste less time
- work harder
- stop procrastinating
- fix my routine
- do better

## ⚡ Energy Log

Use this to track what gives or drains energy. It does not need to be perfect.

### Emoji legend

🟢 **Energizing**

Means the activity gives energy, momentum, clarity, confidence, or makes the day better.

Examples:

- good coding flow
- shipping something concrete
- walk after work
- cleaning room and feeling clearer
- finishing a small annoying task
- direct progress on project
- writing a clear next step
- gym/cardio if it made the day better
- social interaction that left you calmer/better

🟠 **Neutral**

Means necessary or okay. It did not really help, but did not damage the day either.

Examples:

- groceries
- laundry
- basic admin
- short docs reading
- normal school task
- routine cleanup
- replying to messages
- small maintenance task
- normal planning with clear end

🔴 **Draining**

Means the activity takes energy, creates avoidance, stress, guilt, friction, or makes the next work harder.

Examples:

- scrolling too long
- vague docs task
- unclear coding problem with no next step
- arguing / conflict
- messy room making work harder
- too much tool tweaking
- planning without output
- tutorial spiral
- switching tasks too much
- checking AI/tools instead of shipping
- admin that expands into the whole day

### Work examples

- 🟢 coding — good flow, clear task — fix: do this earlier
- 🟢 shipping small feature — concrete output — fix: repeat tomorrow
- 🟠 admin — necessary, okay — fix: batch it
- 🟠 reading docs — useful but not output — fix: limit to 20 min
- 🔴 writing docs — avoided it, vague scope — fix: 20 min only
- 🔴 tweaking setup — felt productive but no output — fix: no setup before output
- 🔴 choosing stack — too much comparison — fix: pick one default and build

### Life examples

- 🟢 walk — clearer head — fix: keep after work
- 🟢 cleaning room — better environment — fix: 10 min daily
- 🟠 groceries — normal task — fix:
- 🟠 laundry — fine — fix:
- 🔴 scrolling — too long after work — fix: blocker after 21:00
- 🔴 messy room — made starting harder — fix: 5 min reset before sleep
- 🔴 late caffeine — harder to sleep — fix: cutoff earlier
